Issue 001: loops, context, and evals

Five useful updates

  1. Agent loops — learn the loop before buying agent automation.
  2. Context engineering — treat context like product infrastructure, not prompt decoration.
  3. Lightweight eval harnesses — move recurring AI workflows from vibe checks to repeatable gates.
  4. Hermes + Obsidian personal OS — a practical capture-to-knowledge workflow worth testing manually first.
  5. MCP servers — useful interface pattern, but only valuable when the connected tool is worth exposing safely.

Three things to try

  1. Map one workflow into observe → act → verify → review.
  2. Write a reusable context brief for one repeated task.
  3. Create three fixtures and assert the final artifact sections.

One concept to learn

Agent loops: the smallest useful mental model for deciding where autonomy belongs and where review gates belong.

One thing to ignore

Ignore generic “top AI tools” lists that do not name a use case, constraint, source, or next action.
